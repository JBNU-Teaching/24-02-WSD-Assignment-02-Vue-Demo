과제 4) 카카오 로그인 구현

이번 4차 과제에서는 2차 과제에서 개발한 front-end framework에서 카카오 로그인을 연동하면 됩니다. 정석적인 방법으로는 Front-End / Back-End 모두 개발해서 진행해야 하지만, 무리가 있을 것으로 판단되어, Front-End에서만 진행하도록 하겠습니다. 또한, 2차 과제에서는 TMDB의 API를 호출할 때, key를 발급 받아야 사용할 수 있던 것 기억하시죠? 2차 과제에서는 이를 로그인 할 때 입력 받도록 구현하였는데, 카카오 로그인을 수행하면 이를 실행할 방법이 없습니다. 이에, 4차 과제에서는 TMDB API key를 사용자로부터 입력받아 사용하는 것이 아니라, .env 파일에 저장하여 사용하도록 하겠습니다. 카카오 로그인을 위해서는 다음 가이드를 참고하시면 됩니다.
  • 카카오 로그인 페이지: https://developers.kakao.com/
  • 카카오 로그인 관련 공식 문서: https://developers.kakao.com/docs/latest/ko/kakaologin/common
  • 카카오 로그인 샘플 예제 (React & Vue는 javascript를 보시면 될 것 같습니다):
    • Javascript: https://devtalk.kakao.com/t/rest-api-node-js/119865
    • Java (Spring Boot): https://devtalk.kakao.com/t/rest-api-java-spring-boot/115579
    • Python (Flask): https://devtalk.kakao.com/t/rest-api-python-flask/134383
  • 개발 언어 및 프레임워크: 2차 과제에서 사용한 프레임워크 및 프로젝트 재사용
  • 만약 2차 과제에서 개발을 많이 못하신 분들은 아래 제가 만든 앱을 활용해서 개발하셔도 됩니다. (감점 없음)
    • https://github.com/JBNU-Teaching/24-02-WSD-Assignment-02-Vue-Demo.git
  • 배포는 아래 중 하나를 선택하여 진행하시면 됩니다. (택 1)
    • Netlify: https://www.netlify.com/
    • Github Pages: https://pages.github.com/
    • JCloud: https://jcloud.jbnu.ac.kr/
  • 개발 환경 분리는 아래처럼 하시면 됩니다.
    • .env-dev, .env-prod 사용 및 .gitignore에 추가하시길 바랍니다. (github에 올리지 않도록 주의하도록 하며, 올라와있으면 0점 처리)
    • ???는 각자 세팅에 맞게 설정하시면 됩니다.
    • .env-dev
      • IP_ADDRESS=localhost
      • PORT=????
      • TMDB_API_KEY=YOUR_API_KEY
    • .env-prod
      • IP_ADDRESS=????
      • PORT=????
      • TMDB_API_KEY=YOUR_API_KEY
    • 이 외에 본인이 생각하시기에 분리가 필요하다고 생각되는 것 2개 이상 추가하셔서 개발하시면 됩니다. (1개 당 15% 감점)

D. 과제 개요

1. 주요 개발 내용

(1) 본 과제에서 개발할 내용은 다음과 같습니다.

  • 카카오 로그인 구현
    • Front-End Application
    • 카카오 로그인과 관련된 모든 로직 구현
    • 로그인 및 회원 정보 조회 검증 로직 구현
  • 카카오 회원 정보 조회 구현
    • 카카오 API를 활용하여 프로필 ID 조회
    • 조회한 회원 이름을 header에 표시
    • 이름 외 조회한 정보 최소 1개 이상을 콘솔에 출력
  • 개발-배포 환경 분리
    • .env 파일을 개발-배포 환경을 나눠 정보 기입
    • package.json에서 서로 다른 명령어로 개발-배포 환경 분리
    • 환경 변수 관리 패키지 (dotenv 등) 사용
  • 기타
    • 카카오 로그인을 위한 API Key 발급
    • 카카오 로그인을 위한 API Key를 .env 파일에 저장
    • TMDB API key를 .env 파일에 저장

(2) 평가 항목은 다음과 같습니다.

  • 카카오 로그인 구현 (로그인 성공)
  • 카카오 로그인 후 main 페이지로 자동 redirect
  • 카카오 로그인 후 회원 정보 조회 및 프로필 이름을 header에 표시
  • 카카오 로그인 후 회원 정보 조회 및 콘솔에 출력
  • 새로고침 시에도 로그인 상태 유지
  • 로그아웃 기능 구현
  • 로그인 상태에 따른 UI 분기 처리 (로그인/로그아웃 버튼 등)
  • 카카오 로그인 실패 시 적절한 에러 메시지 표시
  • API 호출 실패 시 예외 처리
  • 네트워크 오류 발생 시 사용자 피드백 제공
  • .env 파일 사용 여부 (.env-dev / .env-prod)
  • .env 파일에 TMDB API Key 저장 여부
  • .env 파일에 카카오 API Key 저장 여부
  • API 키가 GitHub에 노출되지 않도록 관리
  • package.json에 개발/배포 환경 실행 스크립트가 명확히 구분
  • 토큰을 localStorage 또는 sessionStorage에 적절히 저장
  • .gitignore 파일에 환경 변수 파일 포함 여부
  • 선택한 배포 플랫폼에서의 정상 작동 여부
